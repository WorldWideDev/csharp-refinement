Models

The last part of MVC to introduce is the M -- models. While we often think of models in relation to the database specifically, but in C#, an OOP language, these models can also play other roles. By convention, our models live in the Models folder:

In contrast to the Controllers and Views folders, the Models folder is convention and not required by the framework, so you have more flexibility with how you organize the contents of your Models folder.

By convention, similar to our controllers, we modify our models' namespaces to ProjNamespace.Models.

namespace YourNamespace.Models
{
    public class User
    {
        public string Name { get; set; }
        public string Email { get; set; }
        public string Password { get; set; }
    }
}
