Models

The last part of MVC to introduce is the M -- models. While we often think of models in relation to the database specifically, in ASP they play an even larger role. Let's examine a few central ways models factor into this framework:

  1. A type-safe means of providing data from the Controller to the View (ViewModels)
  2. Submitting forms that map to a defined Model type. With this, we will get slick form validation for free!
  3. A type that will map to tables in your database. Models that bear this responsibility are conventially referred to as Entities

What are Models?

Models are just classes! You will define a class type that will be used as a Model for your desired purpose. A typical model might look like so:

    public class Friend
    {
        public string FirstName {get;set;}
        public string LastName {get;set;}
        public string Location {get;set;}
    }

Now, let's think about how a Model such as this could be utilized in the ways that were enumerated. Well, first, we can imagine a View that wants to describe a single Friend. Maybe we have a website that's all about connecting friends, and has a page that provides details regarding one Friend; namely, his or her first name, last name, and location. We can say this view requires a Friend object be passed to it - so that it is guaranteed to have the data the view requires.

Next, we can imagine a form that might be for creating new Friends in this FriendSite.com! We can use special html attrubutes in our forms that will help us validate each field on our Friend model.

And lastly, it's safe to assume this site would need a database to back it - and with the help of an ORM - we can map our SQL tables, and the columns they possess, to our defined Friend model class.

Models/

By convention, our models live in the Models folder:

Like the Controllers directory, and in contrast to the Views directory, the Models directory is typically named as such, so therefore you have more flexibility with how you organize its contents.

namespace YourNamespace.Models
{
    public class User
    {
        public string Name { get; set; }
        public string Email { get; set; }
        public string Password { get; set; }
    }
}
