Razor View Engine

Now that we're using ASP.NET Core MVC, we have access to the Razor View Engine. The .cshtml file extension we've been using for views actually designates Razor files. Razor allows us to write HTML just as before, but with the added ability to write C# code directly in our views.

To distinguish between regular HTML and C# blocks of code, we use the @ symbol. If we are writing a block of code that involves declaring variables, we use the @ followed by curly braces. Within those curly braces, we can write any C# code we'd like. If we want the value of a variable to print to HTML, we use the @ followed by the variable name (no spaces). If you want to display something to HTML, be sure to wrap this syntax in an HTML tag of some kind.

<body>
    @{
        var StringList = new List<string>{
                                "one",
                                "two",
                                "three",
                                "four"
                            };
        
        foreach(var word in StringList)
        {
            // We can render HTML from within C# code!
            <p>@word</p>
        }
    }
</body>

We can write HTML in our C# block and it will be rendered as the C# evaluates when the page is loaded. Any C# operation or method call can be made outside of a C# code block (@{ }) by leading the operation/method call with the @ symbol. We can nest C# inside HTML inside C# and so on as much as we like:

<body>
    @foreach(var word in StringList)
    {
       <div>
          <p>@word</p>
          @if(word.Length < 4)
          {
             <p>@word is a short word</p>
          }
       </div>
    }
</body>
