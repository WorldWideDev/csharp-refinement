Models with Forms

Overview

One of the main uses for models is to map a model type to a form for user submitted data. You can imagine a user registration form, with fields for FirstName, LastName, etc. and how that would map nicely to a C# class - where text inputs will be string types, number fields int types and so on. When we define a model class that maps directly to the name attributes in a form, we are able to accept an object of that type in the Controller method responsible for handling the request. This would be opposed to accepting a series of string arguments, as we have seen thus far. Consider the following implementation of the DojoSurvey assignment.

    // User.cs (model file)
    public class User
    {
        public string Name {get;set;}
        public string Location {get;set;}
        public string Language {get;set;}
        public string Comment {get;set;}
    }

    // Index.cshtml
        // Note that the input's name attributes match that of the User class's member properties.
    <form action="survey" method="post">
        <label for="Name">Your Name:</label>
        <input type="text" name="Name">

        <label for="Location">Dojo Location:</label>
        <input type="text" name="Location">

        <label for="Language">Favorite Language:</label>
        <textarea name="Language"></textarea>
        <button>Submit</button>
        </form>
