SQL Model Construction and Queries

(If you are feeling confident with MySql Queries and building DB schema you may skip this section!)

At this point, it's past time that we get working with some persistent data! Databases are a major part of every field in programming and this holds especially true for the web. The database we will be working with here will be the Open-Source Relation Database called MySQL(You may have heard of it before :D).

To start out, we first need a way to send queries to our database to build out schemas and tables where we can insert our data. An easy way to do this is to make use of the visual tool MySQL Workbench. MySQL Workbench can be downloaded from the following link if you do not already have it installed on your machine:

https://www.mysql.com/products/workbench/

With MySQL Workbench installed we are going to begin working on building out our ERD diagrams then running queries for the newly created tables.

Why ERD?

ERD is a process of laying out your tables and establishing relationships between them, making your data relational. Just about any data imaginable can be stored in a relational matter, there really isn't anything you can't do using a relational database like MySQL. Later, you will learn non-relational databases where everything is stored in a single table. There are pros and cons for both, but we find it's much easier to go from a relational database to a non-relational database.

To build out our ERD diagrams we should probably make sure we can connect to our Database at all first. There are only a few steps involved at this point, but they are all pretty critical.

  • Start MAMP and Launch your Server
  • If the MySQL Server checkbox is green you should be good there
  • Launching a new instance of MySQL Workbench user or add a new MySQL Connection
  • The connection should be configured to be pointing to localhost and the port MySQL is running on

  • Enter your Username and Password (typically "root") and double click the new connection

  • If all worked out you should now be at a query prompt
