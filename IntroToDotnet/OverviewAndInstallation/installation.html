Install

For this course, we need to first get a few things installed before rolling right into the material. Some of these things might not be used until a little later, but it is nice to get them out of the way upfront.

Installing Visual Studio Code

While it is entirely possible to go through this course using any text editor you want or a full IDE like full Visual Studio we highly recommend installing and working with Visual Studio Code. It is a good in between of the barebones standard text editor and the magic Visual Studio can provide while also being completely cross-platform to boot.

All you need to do to get the latest version of VS Code is go to https://code.visualstudio.com and click the download button.

Installing the 'C# for Visual Studio Code' plugin from OmniSharp

The OmniSharp plugin will give us a ton of extra features when working with C# code in Visual Studio Code -- such as ASP.NET syntax highlighting, early (pre-compile) error detection, code suggestions and completion, and insight into different class functionality. It is easy to add: simply open a new instance of VS Code, find it on the Extensions menu, and install+enable.

MacOS Specific

Before installing the .NET Core SDK on Mac OS you need to have the latest version of OpenSSL set up. The easiest way to get OpenSSL is by using Homebrew. If you do not already have Homebrew you can download and install it by going to their Homepage . Once installed, do the following in terminal... 

brew install openssl
ln -s /usr/local/opt/openssl/lib/libcrypto.1.0.0.dylib /usr/local/lib/
ln -s /usr/local/opt/openssl/lib/libssl.1.0.0.dylib /usr/local/lib/

From there we can download the .NET Core SDK v1.1 for Mac by clicking the link below!

Nuget

With any framework we need to draw in packages that other people have written to supplement our code. Most package managers are designed for use with a specfic language or framework. For Python packages we use pip , for Node we use npm , Ruby has RubyGems , and Microsoft's family of .NET products use NuGet . www.nuget.org hosts all the packages that we will pull in as dependencies. We don't actually have to do any extra work to enable NuGet, it will be used under the hood by ASP.NET Core. Unlike pip or npm we won't be using NuGet from the commandline to install packages one by one. ASP.NET Core lets us write all of our package dependencies in a json file which can then be used to install all of them with a single command.
