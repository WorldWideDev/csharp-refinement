Open Source, Cross-Platform

Microsoft has a history of being an open platform that welcomes third-party contributions on top of those platforms. As mentioned earlier, the CLR supports any programming language that can compile itself into CIL format. Starting at the very beginning with v1.0, the CLR has published a specification of the Common Language Infrastructure (CLI), spelling out what language providers must do in order to work as .NET-supported languages. This has made it possible for more than 20 languages to have some sort of .NET implementation.

What was less common until recently was Microsoft's willingness to allow substitutions of its own platform pieces. Once an unspeakable subject, it is now reasonable to consider Microsoft core products running on other operating systems. For example, the CLI made it possible for the company Xamarin to create an implementation of the .NET Framework for Unix and Linux platforms called Mono. It also enabled the game authoring engine Unity to build C# code that could run on Mac, Linux, or Windows. Microsoft has announced that nearly 1/3 of all Azure virtual machines are running Linux.

OneNote, considered by some to be the "secret weapon in Office," not only runs on MacOS, iOS, and Android, but is also free! while there is a paid premium version, this was a notable indicator that many at Microsoft had changed their mindsets.

In mid-2015, Microsoft announced that it was developing a free, cross-platform source code editor called Visual Studio Code (VS Code). With a plug-in from OmniSharp, it supported debugging, Git, syntax highlighting, and code completion. 

Beyond just being free and cross-platform, Microsoft has also been releasing more and more programming language properties, as open source, such as the F# compiler, the "Chakra" JavaScript engine, the next-generation "Roslyn" compilers for C# and VB, and even the PowerShell scripting environment.

Before the end of 2015, Microsoft had released VS Code as open source as well. What next?!
