Welcome to .NET

By the late 1990's, developers of software for Microsoft platforms could choose from a few different languages. To attract a broad developer base, Microsoft had produced tools to support C++, BASIC, Assembly and even an implementation of Java (J++). However, over time the runtimes of these languages had diverged in both feature coverage and behavior, leading to headaches for developers inside and outside Microsoft. An initiative within the company was formed to create a new model for software developers, and this eventually became the .NET Framework . 

The primary deliverable for this project, which was initially referred to as the Universal Runtime, was a code-generating runtime that could host a number of high-level programming languages, as well as a set of compilers that would reduce those different high-level languages into the same intermediate language (MSIL, often called Common Intermediate Language or CIL). Microsoft believed that with a single runtime they could add emerging languages, and provide features across those languages more consistently , with less effort. They took this consistency further by producing Visual Studio .NET , a unified development environment for all .NET languages. Finally, by separating code generation from the previous compile stages, this runtime could minimize the amount of new code needed to support upcoming hardware platforms. 

The .NET Framework v1.0 was released in 2002, along with Visual Studio .NET and the programming language C# . New versions have been released every 1-2 years since then. 

What's in the .NET?

The .NET Framework consists of a number of significant components. The bottom layer is the Common Language Runtime (CLR). The CLR manages just-in-time compilation, type safety, memory, threads, security, and other services on behalf of applications. All source code in .NET languages are compiled beforehand into CIL, and at run-time, this code is then further compiled, managed, and run within the CLR. So it is accurate to call that source code managed code , and you could think of the CLR as an application virtual machine . 

Sitting above the CLR, there is an extensive collection of libraries. The essential service, called the Base Class Library (BCL), have been included since v1.0. Since that first release in 2002, more libraries have been added with each new version of the framework. These include the following (that you should eventually learn about) and others (WPF, WF, WinForms, etc.):

  • ASP.NET - technology for building web apps
  • Windows Communication Foundation ( WCF ) - technology for service-oriented applications
  • Windows Identity Foundation ( WIF ) - authentication and claims-based identity support
  • Language-Integrated Natural Query ( LINQ ) - adds query syntax directly into C# and VB
  • Entity Framework ( EF ) - an ORM that uses LINQ

Along with the CLR, the framework includes the compilers for languages with built-in support, such as C# and Visual Basic, and more recently, a functional language named F# . Additional languages that are or were previously supported by .NET include implementations of Java, Python, Ruby, Ada and Eiffel (J#, IronPython, IronRuby, A#). All of these featured compilers that produce CIL code consumable by the CLR.
