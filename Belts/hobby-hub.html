For a Red Belt:

  • Login and Registration with Validations
    • Validation messages should display for unsuccessful submissions
    • (Registration) Password with a minimum length of 8 characters
    • (Registration) First Name and Last Name with a minimum length of 2 characters
    • (Registration) Username between 3 and 15 characters
    • (Registration) No duplicate Usernames
    • (Login) Valid Email and Password
  • Dashboard
    • Display all hobbies and number of enthusiasts for each hobby
    • Clicking on the name of a hobby should route to a hobby-details page
  • Hobby Details
    • Display the name and description of shown Hobby
    • Display the names of all users that have added shown hobby
    • Include a button to add shown hobby to user's hobbies
    • PROFICIENCY RELATED FEATURES NOT REQUIRED FOR A RED BELT
  • New Hobby
    • Validation messages should display for unsuccessful submissions
    • Name and description are required fields

For a Black Belt:

  • (Mandatory) Exam is deployed to either Amazon EC2 or Microsoft Azure. IP address to hosted site should be included in a text file.
  • (Mandatory) All Red Belt features.
  • 3/5 of the following features:
    • (Dashboard) Display a leaderboard showing the top hobby(ies) for each proficiency level. "None" if no hobbies have been added.
    • (Hobby Details) Display proficiency level of each listed user. When adding a hobby, have the user select a proficiency level (Novice, Intermediate, Expert).
    • (Hobby Details) A User should not be able to add a hobby more than once.
    • (New Hobby) Hobby name should be unique.
    • (Edit/Update) User should be able to edit/update hobbies. Update form should validate for required fields.
