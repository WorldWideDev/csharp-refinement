Models

One of the powerful tools that ASP.NET Core puts in our hands is the concept of models. Models are special classes that we can use to validate user input and to facilitate making entries in a database. However, we don't need a database in order to use models. In most cases, our models will correspond to our controllers, i.e. UsersController.cs should contain methods that relate to User objects, though there are some exceptions. Our models live in the Models folder:

By convention, we modify the namespace that our models live in to our project namespace followed by .Models.

namespace YourNamespace.Models
{
    public class User
    {
        public string Name { get; set; }
        public string Email { get; set; }
        public string Password { get; set; }
    }
}

This basic user model doesn't have any functionality beyond a normal class yet, but just being able to define classes and use them from our controllers is useful. Classes allow us to add significance to groups of values and attach functionality to them. In the following tabs we'll learn more about the special functionality of Models.

BaseEntity

A common convention when working with ASP.NET Core Models is to define a BaseEntity class that all of our Model classes inherit from. BaseEntity serves as a generic reference that we can use to distinguish our Models from all other classes. We can also add fields or methods to BaseEntity as long as they are used by all of our defined Models.

BaseEntity is commonly implemented simply as an empty abstract class. 

namespace YourNamespace.Models
{
    public abstract class BaseEntity {}
}
