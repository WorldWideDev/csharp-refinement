Deploying with a Database

For our first deployment with a database, we're going to use Azure's new built-in MySQL tool. Follow the regular deployment instructions and then navigate to the "MySQL In APP (preview)" tab as shown. Turn on the in-app Db and save it. Navigate to your deployed page once and the database tools will be activated, then click "Manage" at the top of the MySQL In App tab. If you do not visit your page at least once you will be unable to access the database tools!

If you see the following screen when you attempt to manage your DB:

Then you have not visited your page yet! Navigate to your deployed site, even if it has no content yet.

You can manually create a database within the database admin page, or upload a copy of a local database. To copy one or your existing databases you'll have to first export it into a .sql file. To do so, open up one of your MySQL workbench connections and select "Data Export" under the "Server" tab. Choose the schema/s you want to copy and make sure to check the "Include Create Schema" option. Once the .sql file has been created, return to the Azure database manager.

  1. Choose the .sql file you created.
  2. Upload the schema and copy it to your in-app database.
  3. Use the given IP and Port to configure the database connection string in your code.

Whether you create a new schema or upload one, you will have to configure your connection string to match your in-app database.

{
    "DBInfo":
    {
        "Name": "MySQLconnect",
        "ConnectionString": "server=127.0.0.1;userid=root;password=password;port=YourDbPort;database=YourDbName;SslMode=None"
    }
}

With your connection string set up, simply commit the new changes to your git repo and push it to Azure!
